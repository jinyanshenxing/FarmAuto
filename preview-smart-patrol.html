好友智能巡田逻辑

基于作物成熟时间的按需巡查策略，大幅减少无效 API 请求，降低被检测风险

📊 改造前后对比

❌ 改造前：全量遍历

🔄每 10 秒触发一次好友巡查周期
👥每次遍历全部好友，逐个进入农场
📡50 个好友 = 每周期 100+ 次 API 请求（进入 + 离开）
⚠️大量无效请求（作物未熟也要进入查看）
🚨高频访问容易触发官方风控检测

✅ 改造后：智能调度

🧠首次访问后记录每个好友的最早成熟时间
⏰作物未熟的好友自动跳过，到期才访问
📉API 请求量降低 70%~90%
🎯有可偷/可帮好友仍立即访问（优先级不变）
🛡️访问模式更像真人操作，有效规避检测
🔄 新巡查流程
📋
1. 拉取好友列表
调用 getAllFriends()（单次 API），获取所有好友的摘要信息（可偷数量、缺水/有虫等状态）
⭐
2. 优先好友：有可操作内容 → 立即访问
好友列表显示 stealNum > 0 或有缺水/虫草 → 放入优先队列并清除旧调度，确保第一时间访问
⏳
3. 普通好友：检查调度表 NEW
查询 friendNextVisitMap：
• 有调度且未到期 → 跳过本轮（作物还没熟）
• 调度已到期或首次访问 → 加入访问队列
关键优化点：避免反复进入未成熟的好友农场
🏠
4. 进入好友农场 → 执行操作
对队列中的好友依次执行：帮助（浇水/除草/除虫）→ 偷菜 → 捣乱
📅
5. 访问后设置下次调度 NEW
• 执行了操作 → 清除调度，下次正常巡查
• 无可操作且有未成熟作物 → 计算最早成熟时间，设置下次访问
• 最长延迟 1 小时（兜底）、最短 30 秒、成熟后多等 10 秒（模拟人类反应）
📝
6. 输出巡查日志
汇总日志包含：实际访问人数、跳过人数、最近一位好友的成熟时间

🛡️ 安全策略说明
🎲 随机化缓冲
作物成熟后额外等待 10 秒再访问，避免精准定时的机器人行为特征
📏 延迟上限
最长延迟 1 小时，即使作物需要数小时才熟，也会定期检查（防止状态过期）
🔄 优先级覆盖
好友列表如果显示有可偷/可帮内容，立即清除调度并优先访问，不会错过机会
🔃 每日重置
每日 0 点自动清空所有调度，确保新的一天从完整巡查开始
